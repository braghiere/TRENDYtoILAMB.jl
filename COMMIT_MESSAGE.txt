feat: Comprehensive TRENDY to ILAMB conversion with robust time handling

Major updates to handle diverse TRENDY v13 time encodings and generate
ILAMB-compliant outputs for 22 models and 19 variables.

## Core Changes

### Time Handling (src/time.jl)
- Fixed convert_time_to_days to properly handle different reference years
- Changed from: (t - reference_year) * 365
- Changed to: (reference_year + t - 1850) * 365
- Fixes date extraction for CABLE-POP (1700 reference) and other models
- Added helper functions for time bounds creation
- Support for days, months, years, hours since various dates

### Conversion Logic (src/converters.jl)
- Implemented safe attribute access with fallback defaults
- Added model-specific handlers (CARDAMOM month indices)
- Datetime-based date extraction for accurate filename dates
- ILAMB-compliant filename format: {var}_Lmon_ENSEMBLE-{model}_historical_r1i1p1f1_gn_{YYYYMM}-{YYYYMM}.nc
- Simplified output structure: direct to {MODEL}/ folders
- Time bounds handling for both dimension orderings (2×n and n×2)

### Batch Processing (examples/convert_all_trendy.jl)
- Support for 19 ILAMB variables (carbon fluxes, hydrology, pools, fire, forcing)
- Added tas, pr, rsds forcing variables
- Comprehensive error handling and statistics tracking
- NetCDF validation before conversion
- Data verification after conversion
- Configurable output directory
